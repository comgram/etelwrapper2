◆ esc_create_offline()

int esc_create_offline ( ESC **  esc,
int  nb_devices,
ESC_OFFLINE_DEVICE  present_devices[] 
)

Create a ESC object which will allow to compile a sequence file offline, without any connection to hardware.

Warning: Allocate Heap memory

This required that the sequence is written in increment (no ISO value can be used). The user must also specified the configuration (device type and device firmware) for which the file must be compiled. This is requeired to check the commands and registers aliases, as well as the command parameter validity.

Parameters
escthe address of a pointer to the ESC object
nb_devicesthe number of devices present in the table present_devs
present_devicesa table which contains all the devices the sequence will refer to. Case 1: The sequence is a sequence which will be downloaded into an UltimET: UltimET and all devices referred in the sequence must be specified. (This is the case if the UltimET sequence sends commands or access AccurET registers). If the devices are specified with their number (example X10.15), axis 15 and its corresponding type must be specified in the device table. If the devices are specified with .!, at least one device of each used type (AccurET 400, AccurET 300, etc) must be specified. Case 2: The sequence is a sequence which will be downloaded into a specific AccurET-type and number: (In the sequence, only instruction like X10.9 are present) The AccurET corresponding number with its type must be specified. Case 3: The sequence is a sequence which will be downloaded into a specific AccurET-type but with unknown number: (In the sequence, only instruction like X10.! are present) At least one AccurET with corresponding type must be specified. Case 3: The sequence is a sequence which will be downloaded into many AccurET typs with unknown number: (In the sequence, only instruction like X10.! are present) At least, one AccurET of each AccurET typ must be specified.
Returns
a negative value if an error occurs, 0 otherwise
ESC_EBADLABEL label number out of range
ESC_EBADPARAM bad parameter
ESC_EBADSEQVERSION Version syntax error (Syntax: [n]n.nnc)
ESC_EBADSTATE bad state

int  code 
)

Convert an error code into a textual error message.

Warning: Allocate Heap memory

Each function of the ESC library returns an integer (int) which represents the error code. If the operation ends with success, the function returns 0, otherwise it returns the number of the error represented by a negative number. This function enables the conversion of this error code into a textual error message in the form of a character string.

Parameters
esca pointer to the compiler base structure
codeThe error code returned by a esc lib function.
Returns
A pointer to a string containing the error message.
