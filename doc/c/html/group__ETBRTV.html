◆ etb_rtv_watchdog()

int etb_rtv_watchdog ( ETB *  etb,
int  rtv_id,
ETB_RT_CHECKPOINT_TYPE  watchdog_type,
ETB_WATCHDOG_REACTION  watchdog_reaction,
double  watchdog_time 
)

Activate or deactivate the RTV watchdog on UltimET-Advanced product.


The watchdog let the UltimET check that the RTV cyclic user function is processed in the imparted time. If no, the UltimET will fall into error. !!! Only available on UltimET Advanced PCIe/CSM connection !!!
!!! For Ultimet Light, use etb_irq_watchdog !!!
On UltimET Advanced VxWorks CSM, the watchdog has a granularity of TransnET cycle 100us, therefore watchdog parameter cannot be less than 0.1, except 0 for disabling
On UltimET Advanced PCI or FreeRTOS CSM, the watchdog has a granularity of 10 ns, therefore watchdog parameter cannot be less than 0.00001, except 0 for disabling

Parameters
etbpointer to etel bus structure
rtv_idthe RTV id (Actually only 0)
watchdog_typethe check point type where the watchdog will be active
  • ETB_RT_CHECKPOINT_TYPE_IRQ: Watchdog on IRQ acknowledge time
  • ETB_RT_CHECKPOINT_TYPE_USRAPP: Watchdog on USRAPP time
  • ETB_RT_CHECKPOINT_TYPE_EDI: Watchdog on EDI total time
watchdog_reactionthe kind of reaction if a watchdog is detected
  • ETB_WATCHDOG_REACTION_NONE: No reaction at all
  • ETB_WATCHDOG_REACTION_WARNING: Not usable
  • ETB_WATCHDOG_REACTION_ERROR: UltimET advanced falls in error
watchdog_timethe time in ms. 0 will disable the check
Returns
a negative value if an error or a timeout has occured

◆ etb_start_rtv_handler()

int etb_start_rtv_handler ( ETB *  etb,
int  nr,
int  rate,
int  delay,
ETB_RTV_HANDLER  handler,
void *  user 
)

Start a real-time handler.

!!! Only available on AccurET family devices connected through TransnET The specified handler will be called at each UltimET cyclic interrupt (if rate = 1). By specifying rate different than 1, the function will be called each "rate" cyclic interrupt. Rate of 0 will disable the cyclic interrupt and thus disable the function call. The UltimET cyclic interrupt will occured each 100 us on standard configuration (This can be modified). The cyclic interrupt can be delayed by "delay" * interrupt. Three cyclic interrupt can be used. Specify which one with nr parameter.

Parameters
etbpointer to etel bus structure
nrthe cyclic interrupt number "0 .. 2"
ratethe rate of the cyclic interrupt
  • 0 : disable cyclic interrupt "nr" and thus disable handler call
  • 1 : cyclic interrupt "nr" and handler call appears each 100 us (by standard TransnET configuration>
  • 2 : cyclic interrupt "nr" and hanlder call appears each 200 us (by standard TransnET configuration>
  • ...
delaythe delay between real UltimET cyclic interrupt and generated cyclic interrupt
  • 0 : no delay
  • 1 : 30 ns delay(by standard TransnET configuration>
  • 2 : 60 ns delay(by standard TransnET configuration>
  • ...
handlerthe user function which will be called. be aware that this function call will suspend ETB TransnET message handling. Thus, this function should avoid consuming too much time.
usera pointer to user value which will be returned as-is at each handler call.
Returns
a negative value if an error or a timeout has occured
