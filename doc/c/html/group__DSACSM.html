◆ dsa_customer_software_module_command_interface_set_function()

int dsa_customer_software_module_command_interface_set_function ( DSA_MASTER *  master,
unsigned int  fctIdx,
DSA_CUSTOMER_SOFTWARE_MODULE_COMMAND_INTERFACE_FUNCTION  fct,
void *  user_param 
)

Attach the user function the the specified function index.

!!! Only available for Customer Software Module running on UltimET Advanced product

After that, a user will be able to start the specified function by calling *JMP=fctIdx,threadIdx
By calling *JMP=fctIdx,threadIdx, the function will then be executed inside the thread referenced by threadIdx
If the specified thread already executes a function, the first function will be ended first, and replaced by the new one
The function prototype has a first parameter allowing to stop the function:

typedef struct {
    ebool               stop_requested;         // TRUE if the function must stop
} DSA_CUSTOMER_SOFTWARE_MODULE_COMMAND_INTERFACE_SYNCHRO;

If the function is a long-term loop, it should be able to stop by checking the value pointed by stop_requested field of first function parameter.
This value pointed by stop_requested will be set to TRUE when the user enters a END=threadIdx

Parameters
masterpointer to UltimET Advanced master
fctIdxthe index referencing the function (must be less than max_fct specified by function etb_csm_cmd_interface_create)
fcta pointer to the user function which will be called when user enter JMP=fctIdx,threadIdx
user_parama pointer to the user parameter which will be passed "as is" to the function
Returns
a negative value if an error or a timeout has occured

◆ dsa_wait_customer_software_module_s()

int dsa_wait_customer_software_module_s ( DSA_MASTER_BASE *  obj,
dword  what,
long  timeout 
)

Waits for a specific state of customer software module (Command:WAITCSM) - synchronous.

Available on UltimET Advanced ONLY.

Parameters
objthe object pointer
whatthe status of csm to wait for: the OR-ed value of following definition:
  • DSA_CSM_WAIT_FOR_CSM_STARTED Wait for CSM started in remote controlled (Release mode controlled by Application (STARTCSM, JMP, END)
  • DSA_CSM_WAIT_FOR_CSM_CONNECTION_ESTABLISHED Wait for CSM connection to UA established (CSM has done first dsa_open_u)
  • DSA_CSM_WAIT_FOR_CSM_CORE_RUNNING Wait for CSM core (3) running (CSM is started by STARTCSM or SDK)
timeoutthe timeout value in ms or
  • DSA_DEF_TIMEOUT a value depending of the used communication bus (generally 1 or 2 seconds)
  • INFINITE infinite timeout
Returns
a negative value if an error or a timeout has occured
