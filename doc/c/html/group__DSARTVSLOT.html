Assign a register to a slot (Command:ASW=reg,slot,[slot]) - asynchronous.

After this command, the device specified register value will be copied into the specified slot If the slot is a 32-bit slot, the specified register value (which must be either int32 or float32 register) will be copied into the slot at each TransnET cycle.

After this command, the slot value will be copied into the specified of all devices present in grp at each TransnET cycle.

If the slot is a 32-bit slot, its value will be copied into M or MF depending of the register typ.
The register index allowed range is:

  • AccurET version < 3.13 : M(MF)450 - M(MF)457
  • AccurET version >= 3.13 : M(MF)450 - M(MF)465
  • All UltimET version except 3.13: M(MF)450 - M(MF)465
  • UltimET version 3.13: M(MF)310 - M(MF)373
  • UltimET version >= 3.16: Depending of setting (K???) M(MF)450 - M(MF)465 or M(MF)310 - M(MF)373
  • UltimET Advanced: M(MF)310 - M(MF)373 If the slot is a 64-bit slot, its value will be copied into ML or MD depending of the register typ.
    The register index allowed range is:
  • AccurET version < 3.13 : ML(MD)450 - ML(MD)453
  • AccurET version >= 3.13 : ML(MD)450 - ML(MD)457
  • All UltimET version except 3.13: ML(MD)450 - ML(MD)457
  • UltimET version 3.13: ML(MD)310 - ML(MD)341
  • UltimET version >= 3.16: Depending of setting (K???) ML(MD)450 - ML(MD)457 or ML(MD)310 - ML(MD)341
  • UltimET Advanced: ML(MD)310 - ML(MD)341

  • UltimET version 3.13: ML(MD)310 - ML(MD)341
