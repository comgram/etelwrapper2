Start the acquisition and wait that all traces are available (Command:ZFT+ZTE) - asynchronous.

Depending of the trace configuration, the synchronisation will be done by the master (MASTER), by the device, or by the function itself.

  • If all devices specified by acquisition constructor are on same bus and if this bus has a master, the synchronisation will be done by the master.
  • If only a single device is specified, the synchronisation will be done by the device (K120=0).
  • If many devices are specified, if they share not the same bus or if there is no master on the bus, synchronisation will be done by the PC.

Configure the acquisition frequency.

Must be called AFTER dsa_acquisition_..._trigger

Deprecated:
since EDI-4.24A. Use one of the specific frequency configuration functions instead.
Parameters
acqthe acquisition object
nb_pointsthe number of points to acquire (must be greater than 1)
total_timethe total time of the acquisiton in seconds (must be greater than 0)
synchro_modethe synchronisation mode of the acquisition
  • DSA_ACQUISITION_SYNCHRO_MODE_NONE the given number of points will be respected, the total_time will be respected for the best. The acquisition frequency will be a multiple of the highest frequency of the devices' traces (MLTI trace or PLTI trace), but limited by the highest possible frequency of the device (100 us for UltimET).
    Highest frequency possible values are:
    • the PLTI frequency (50 us) if the acquisition is limited to AccurET devices and at least 1 PLTI trace
    • the MLTI frequency (400 us) if the acquisition is limited to AccurET devices and have ONLY MLTI trace
    • the UltimET frequency (100 us) if the acquisition contains at least 1 UltimET trace
      The acquisition frequency is then computed to obtain a multiple of this highest frequency which will best fit the specified total_time. The total acquisition time is then re-computed using this acquisition frequency and the specified number of points.

Start a upload of a trace (Command:SUT) - synchronous.

This function will start an upload of traces values. To really upload the trace datas, several call to dsa_upload_data_s must be done. When dsa_upload_data_s return 0 bytes, this means that the selected trace has been completely uploaded. It is recommended to call this function until 0 byte is returned. This will properly terminate the upload data process.
