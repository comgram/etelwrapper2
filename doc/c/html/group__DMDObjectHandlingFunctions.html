Create the meta-data structure.

Warning: Allocate Heap memory

For firmware version < 3.00, the structure will be created from dmd static data.
For firmware version >= 3.00, the structure can be created ONLY if a same version of drive has been connected during the session.
If it is not the case, you have to call dmd_simulation_create which will create a meta-data structure from the pool of FIRMWARE.

Parameters
rdmdaddress of a pointer to the meta-data structure
d_prodthe product number of the drive
d_verthe version number of the drive
x_prodthe product number of the extension card
x_verthe version number of the extension card
Returns
0 or a negative value on error:
DMD_EBADCOMPRESSTOOL Unhandled compress tool
DMD_EBADDRVPROD bad drive product
DMD_EBADDRVVER bad drive version
DMD_EBADEXTPROD bad extension card product
DMD_EBADEXTVER bad extension card version
DMD_EBADPARAM bad parameter
DMD_EINTERNAL internal error
DMD_EMETAHEADER bad metadata header
DMD_EMETAHEADERVER bad metadata header version
DMD_EMETAPARSER Metadata parsing error
DMD_ENOFWINPOOL Firmware not found in FW pool
DMD_ENOTIMPLEMENTED function not implemented
DMD_EOBSOLETE function is obsolete
DMD_ESYSTEM system error
DMD_EUNZIP Unable to unzip metadata buffer

Create a simualtion meta-data structure.

Warning: Allocate Heap memory
This function uses 7zip feature: License

If the DMD structure exists, take this data from existant structure. This is the case for firmware < 3.00 or for device with metadata which has already been connected during the session.
If the DMD structure does not exists, the metadata are extracted from firmware pool passed as parameter. On WINDOWS systems, if this firmware pool parameter is NULL, the function will look for the sub-directory "fw" under the directory of the current EDI dll.

Parameters
rdmdaddress of a pointer to the meta-data structure
fw_poolthe directory which contain a list of available ETEL-firmware or NULL if automatical search(Windows only)
d_prodthe product number of the drive (if <= 0, simulate an AccurET 400)
d_verthe version number of the drive (if = 0, simulate the last known product)
x_prodthe product number of the extension card
x_verthe version number of the extension card
Returns
