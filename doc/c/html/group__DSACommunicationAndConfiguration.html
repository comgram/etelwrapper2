◆ dsa_open_u()

int dsa_open_u ( DSA_DEVICE *  dev,
char_cp  url 
)

Open a dsa device object.

Warning: Allocate Heap memory

Parameters
deva pointer to the drive structure
urlthe port used to establish the communication
Depending of driver used, url parameter has different syntaxes.
The general syntax is :
etb:<communication bus>:<axis number> where
  • <communication bus> can be:
    • ETN://<hostname>[:<port>][,<flags>]
      • Predefined ports:
        • ULTIMET-TCP/IP : 1129, 1128 or 1127
        • ACCURET through TCP/IP : 1129
      • Available <flags>:
        • T=<keep-alive>:
          • For UltimET Light-TCP-IP, UltimET advanced TCP/IP and AccurET TCP/IP: T=<keep-alive> time in 500 ms packs or -1 to disable
            The communication between EDI and Device is kept by sending records each <keep-alive> ms.
            Both EDI & device checks the communication activity and if it detects that after 3 times the <keep-alive> time no record is transmitted, it will generate an error and disconnect. Setting -1 as keep alive will disable the check
        • x:
          • For UltimET light and UltimET advanced TCP-IP: The AccurET drives will be rebooted before opening communication
    • USB:
    • ULTIMET[ DEVIDX=<n>][,<flags>]:
      • DEVIDX=<n> : indicated the UltimET index in the case where many UltimET are present in the PCI slots
      • Available <flags> (can be cumulated for example ,rx):
        • r: reset UltimET. the UltimET processor is resetted and firmware restarted.
        • x: The AccurET drives will be rebooted before opening communication
    • UA[ DEVIDX=<n>][,<flags>]:
      • DEVIDX=<n> : indicated the UltimET index in the case where many UltimET are present in the PCI slots
      • Available <flags> (can be cumulated for example ,rx):
        • x: The AccurET drives will be rebooted before opening communication
        • T=<keep-alive> time in 500 ms packs or -1 to disable
          The communication between EDI and Device is kept by sending records each <keep-alive> ms.
          Both EDI & device checks the communication activity and if it detects that after 3 times the <keep-alive> time no record is transmitted, it will generate an error and disconnect. Setting -1 as keep alive will disable the check
    • FREERTOS_SHM[,<flags>]:
      • Available <flags> (can be cumulated for example ,rx):
        • x: The AccurET drives will be rebooted before opening communication
        • T=<keep-alive> time in 500 ms packs or -1 to disable
          The communication between EDI and Device is kept by sending records each <keep-alive> ms.
          Both EDI & device checks the communication activity and if it detects that after 3 times the <keep-alive> time no record is transmitted, it will generate an error and disconnect. Setting -1 as keep alive will disable the check
  • <axis number> is a number from 0 to 62 for a drive, or * for a
    Remark:
    When opening communication on several devices, it is mandatory to use the same URL for each device (except the axis number).
    For example, if URL etb:UltimET,rx:* is used to open communication on UltimET, URL etb:UltimET,rx:0 must be used to open communication on AccurET 0.
    Examples :
  • opens communication on drive 2 connected on host ETND) with IP address 172.22.10.103 using port 1131 (see port.properties of host)
    • dsa_open_u(drv, "etb:ETN://172.22.10.103:1131:2");
  • opens communication on drive 3 connected via TCP/IP whose drive has IP address 172.22.246.10
    • dsa_open_u(drv, "etb:ETN://172.22.246.10:3");
  • opens communication on drive 5 connected via TCP/IP whose drive has IP address 172.22.246.10 and disabling KEEP-ALIVE
    • dsa_open_u(drv, "etb:ETN://172.22.246.10,T=-1:5");
  • opens communication on drive 4 connected via USB (AccurET only)
    • dsa_open_u(drv, "etb:USB:4");
  • opens communication on drive 5 connected via UltimET
    • dsa_open_u(drv, "etb:ULTIMET:5");
  • opens communication on drive 5 connected via the first UltimET
    • dsa_open_u(drv, "etb:ULTIMET DEVIDX=0:5");
  • opens communication on drive 6 connected via the second UltimET
    • dsa_open_u(drv, "etb:ULTIMET DEVIDX=1:6");
  • opens communication on drive 2 connected on host ULTIMET-TCP/IP with IP address 172.22.10.103 using port 1129 (see port.properties of host)
    • dsa_open_u(drv, "etb:ETN://172.22.10.103:1129:2");
  • opens communication on drive 2 connected on host ULTIMET-TCP/IP with IP address 172.22.10.103 using port 1129 and disabling keep-alive (see port.properties of host)
    • dsa_open_u(drv, "etb:ETN://172.22.10.103:1129,T=-1:2");
  • opens communication on drive 2 connected on host ULTIMET-TCP/IP with IP address 172.22.10.103 using port 1129 and reset drives (see port.properties of host)
    • dsa_open_u(drv, "etb:ETN://172.22.10.103:1129,x:2");
Returns
