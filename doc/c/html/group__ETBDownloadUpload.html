◆ etb_etcom_download_firmware_ex()

int etb_etcom_download_firmware_ex ( ETB *  etb,
eint64  axis_mask,
eint64  erase_mask,
char *  firmware,
ebool  md5_check,
int *  nb_downloaded_block,
void(ETB_CALLBACK *user_fct)(int, int, int, int, void *)  ,
void *  user 
)

Download specified firmware into specified axis.

Warning: Allocate Heap memory
This function uses 7zip feature: License
This function upgrades your firmware: License

For all blocks in .far file, the function checks if the block must be downloaded using the md5 algorithm. If the block must be downloaded, it is stored in a list. After checking all blocks, the function downloads the blocks stored in the list.
If the bus contains a drive in boot mode which must be downloaded, the block is downloaded without checking the md5.

This function first unzip the specified filename program (provided with EDI package).

  • On Windows, 7z.exe and 7z.dll (provided with EDI) must be present in you PATH environment variable.
  • On RTX & VxWorks-5.4, this functionnality is not available.
    Before using this function, use etb_change_boot_mode function to set the drives or ultimet in the good mode :
    • To download into a ultimet or a drive directly connected through TCP/IP, use
      etb_change_boot_mode(etb, ETB_BOOT_MODE_DIRECT);
    • To download into a drive directly connected through USB, use
      etb_change_boot_mode(etb, ETB_BOOT_MODE_BRIDGE);
    • To download into a drive via a UltimET, use
      etb_change_boot_mode(etb, ETB_BOOT_MODE_BRIDGE);
    • If the directly connected drive cannot be opened in RUN mode because he has download key, use USB connection and open it directly in good mode using
      etb_open(ETB *etb, driver, ETB_FLAG_BOOT_BRIDGE, baudrate, timeout).
      After download, use :
    • etb_change_boot_mode (etb, ETB_BOOT_MODE_RUN).
    Parameters
    etbpointer to etel bus structure
    axis_mask: the mask of the axis to download for example, if the specified firmware is a AccurET-400 firmware, all AccurET-400 axes specified in mask will be downloaded. If no AccurET-400 are specified in mask, function will do nothing and return 0
    erase_mask: ORed values of what must be erased
    • ETB_ERASE_PARAMETERS: Erase parameters
    • ETB_ERASE_SEQUENCES: Erase sequences or customer software module
    firmware: The firmware file name (.far) to download
    md5_checkFALSE: the download will be done without checking the MD5, and then will force the download of all blocks, even if already up-to-date
    nb_downloaded_blocka pointer on a variable which will receive:
    • -1: No blocks have been downloaded nor erased. The MD5 algorithm has detected that all blocks of the firmware are up-to-date.
    • 0: No blocks have been downloaded nor erased. Function has detected that no selected device are compatible with the specified firmware.
    • >0: The number of downloaded or erased blocks.
    user_fct: a user function which will be called each time 20 kbytes have been downloaded This function has 5 parameters:
    • the block number or -1 : Indicates that the MD5 analysis is in progress 0 : Indicates that the erasing of parameter and/or sequences blocks is in progress
    • the total number of blocks which will be downloaded
    • the current block current downloaded size
    • the current block size
    • a user parameter which will be passed as is If NULL is passed, no function will be called
    usera pointer to user value which will be returned as-is at each handler call.
    Returns
    a negative value if an error or a timeout has occured
